Welcome to the UVF IT infrastructure blueprint. Coolify transforms your bare Contabo server into a self-hosted Platform-as-a-Service (PaaS) — giving you the deployment power of platforms like Heroku or Railway, completely on your own hardware. It manages Docker networking, SSL certificates, environment variables, and service orchestration from a single clean dashboard, with zero monthly platform fees.

Platform Architecture
1
Prerequisites

What You Need Before Starting

Make sure the following are in place before running a single command. Skipping any of these is the most common reason Coolify installs fail or the dashboard is unreachable.

Min. CPU
2 vCPU cores
Min. RAM
2 GB RAM
OS
Ubuntu 22.04 LTS
Disk
20 GB+ SSD
Access
Root SSH login
Ports needed
22, 80, 443, 8000
  • Contabo VPS: Get a "Cloud VPS 1" at contabo.com. Choose Ubuntu 22.04 as the OS during checkout. Contabo will email you the server's IP address and root password within minutes of activation.
  • SSH client: On Mac/Linux, your Terminal is ready. On Windows, use Windows Terminal or PuTTY.
  • A domain (optional but recommended): Coolify can auto-issue SSL certificates for a real domain (e.g., coolify.uvfarms.in). If you skip this, you'll access it via raw IP initially.
2
Connect

SSH In & Update the Server

Open your terminal and connect to your freshly provisioned Contabo server as the root user.

A

Connect via SSH

Terminal (your machine)
ssh root@YOUR_SERVER_IP

The first time you connect, SSH will display a fingerprint warning and ask you to confirm. Type yes and press Enter, then enter your root password.

B

Update the Package List & Apply Security Patches

Run these two commands separately. The first refreshes the list of available packages; the second upgrades only security-critical packages to avoid breaking anything.

Bash (VPS)
apt update
apt upgrade -y

Fresh Server Only — Don't Run apt upgrade on Live Production

On a brand new VPS these commands are safe and recommended. On an existing server with running services, apt upgrade -y can silently restart daemons or change library versions. Always review upgrade notes on production machines.

3
Install

Install Coolify with One Command

Coolify provides an official install script that handles everything — it installs Docker, sets up internal networks, pulls the Coolify container images, and starts the dashboard service automatically.

Bash (VPS)
curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash

This process takes 2–5 minutes depending on your server's download speed. You will see a stream of progress output. Wait until it prints a completion message — do not interrupt it.

What the Script Does Under the Hood

The installer: (1) installs Docker Engine if not present, (2) creates a dedicated coolify Docker network, (3) pulls the Coolify, Traefik, and PostgreSQL images, and (4) starts all services as Docker containers managed by a docker-compose.yml at /data/coolify/source/.

4
Firewall

Open the Required Ports (UFW)

Ubuntu 22.04 ships with UFW (Uncomplicated Firewall) which may be blocking port 8000 — the port Coolify's dashboard listens on initially. Run the following to allow access:

Bash (VPS)
# Allow SSH (critical — do this first or you'll lock yourself out)
ufw allow 22/tcp

# Allow Coolify initial dashboard port
ufw allow 8000/tcp

# Allow HTTP and HTTPS for your deployed apps and future SSL
ufw allow 80/tcp
ufw allow 443/tcp

# Enable the firewall
ufw --force enable

# Verify all rules are in place
ufw status

Always Allow Port 22 Before Enabling UFW

If you run ufw enable without allowing port 22 first, your SSH connection will be permanently blocked and you will need to use Contabo's VNC rescue console to recover access.

5
Access

Open the Dashboard & Create Your Account

Once the installation is complete and your firewall ports are open, navigate to the Coolify dashboard in your browser using your server's IP address and port 8000.

Browser Address Bar
http://YOUR_SERVER_IP:8000

You will see the Coolify onboarding screen. Fill in your admin email and create a strong password. This first account is the root admin — keep these credentials safe.

A

Verify all services are running

Back in your SSH session, confirm all Coolify containers are up:

Bash (VPS)
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

You should see containers named coolify, coolify-db, coolify-redis, and coolify-proxy — all with status Up.

B

Point a subdomain for permanent HTTPS access (recommended)

Using a raw IP + port 8000 is fine for setup but not for daily use. In your DNS provider (Cloudflare), create an A record:

DNS Record (e.g. Cloudflare)
Type:    A
Name:    coolify          # becomes coolify.uvfarms.in
Value:   YOUR_SERVER_IP
Proxy:   DNS only (grey cloud) — NOT proxied

Then inside Coolify's dashboard: go to Settings → Instance → Domain and enter https://coolify.uvfarms.in. Coolify will auto-provision a Let's Encrypt SSL certificate via Traefik.

Disable Port 8000 After Setting Up Your Domain

Once your domain + HTTPS is working, run ufw delete allow 8000/tcp to close the raw port. All traffic will then go through Traefik on port 443 — far more secure.

Your Self-Hosted PaaS is Live 🎉

You now have a fully operational Coolify instance on your Contabo VPS. From the dashboard you can deploy Python apps, Node.js APIs, Postgres databases, Redis instances, and more — all with one-click deployments, automatic SSL, and zero per-app monthly fees. Connect it to your GitHub repo for push-to-deploy CI/CD.
