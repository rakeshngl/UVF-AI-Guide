System Architecture
1
Prerequisites

The Setup — What You Need

Before we write code, gather your tools. Think of this as getting your ingredients ready before cooking.

  • A Gmail Account: Create a dedicated Gmail account (e.g., careers@yourdomain.com). You will need an App Password — not your regular Gmail password — to allow the script to authenticate securely. Also enable IMAP in Gmail Settings → See All Settings → Forwarding and POP/IMAP.
  • Cloudflare Account (Free): Sign up at cloudflare.com, navigate to Workers & Pages → AI. Grab your Account ID and create an API Token with "Workers AI" permission.
  • A Contabo VPS: Go to contabo.com and get a "Cloud VPS 1". Choose Ubuntu 22.04 as your OS. They will email you an IP address and root password.
  • Local machine: Python 3.11+ installed, and a code editor (VS Code recommended).

Use a Gmail App Password — Not Your Main Password

Go to your Google Account → Security → 2-Step Verification → App passwords. Generate a 16-character password specifically for this script. Never paste your real Gmail password into environment variables.

2
Code

The Python Brains — agent.py

Create a folder called ai-agent on your computer, and inside it create agent.py with the code below. Read the inline comments carefully — key bug fixes are noted.

Python — agent.py 
import imaplib
import email
import smtplib
from email.message import EmailMessage
import requests
import os
import time

# --- CONFIGURATION (Loaded from server environment variables) ---
EMAIL_ADDRESS  = os.getenv("EMAIL_ADDRESS")
EMAIL_PASSWORD = os.getenv("EMAIL_PASSWORD")   # Gmail App Password
CF_ACCOUNT_ID  = os.getenv("CF_ACCOUNT_ID")
CF_API_TOKEN   = os.getenv("CF_API_TOKEN")

# Cloudflare AI endpoint — Meta Llama 3 (8B) via Workers AI
AI_URL = f"https://api.cloudflare.com/client/v4/accounts/{CF_ACCOUNT_ID}/ai/run/@cf/meta/llama-3-8b-instruct"


def analyze_candidate(email_text):
    """Sends email text to Cloudflare AI for screening."""
    headers = {"Authorization": f"Bearer {CF_API_TOKEN}"}

    system_prompt = """You are an expert tech recruiter. Read the candidate's email.
    If they mention experience with Python, Docker, AND Git, reply ONLY with: MATCH
    Otherwise, reply ONLY with: NO_MATCH"""

    payload = {
        "messages": [
            {"role": "system", "content": system_prompt},
            {"role": "user",   "content": email_text}
        ]
    }

    print("Asking Cloudflare AI...")
    response = requests.post(AI_URL, headers=headers, json=payload).json()
    return response['result']['response'].strip()


def send_interview_invite(to_email):
    """Sends an interview invitation email to a matched candidate."""
    msg = EmailMessage()
    msg.set_content(
        "Hi!\n\nWe loved your profile. Please pick a time for an interview here:\n[Calendly Link]\n\nBest regards,\nUVF IT Recruitment Team"
    )
    msg['Subject'] = 'Interview Invitation — Software Developer'
    msg['From']    = EMAIL_ADDRESS
    msg['To']      = to_email

    with smtplib.SMTP_SSL('smtp.gmail.com', 465) as server:
        server.login(EMAIL_ADDRESS, EMAIL_PASSWORD)
        server.send_message(msg)
    print(f"✓ Sent invite to {to_email}")


def check_inbox():
    """Logs into Gmail via IMAP and processes all unread emails."""
    try:
        mail = imaplib.IMAP4_SSL('imap.gmail.com')
        mail.login(EMAIL_ADDRESS, EMAIL_PASSWORD)
        mail.select('inbox')

        _, selected_emails = mail.search(None, 'UNSEEN')

        for num in selected_emails[0].split():
            _, data = mail.fetch(num, '(RFC822)')
            raw_email = data[0][1]
            msg = email.message_from_bytes(raw_email)

            sender = msg['From']
            print(f"\nProcessing email from: {sender}")

            email_body = ""
            if msg.is_multipart():
                for part in msg.walk():
                    if part.get_content_type() == "text/plain":
                        email_body = part.get_payload(decode=True).decode()
                        break  # stop at first text/plain part
            else:
                email_body = msg.get_payload(decode=True).decode()

            ai_decision = analyze_candidate(email_body)
            print(f"AI Decision: {ai_decision}")

            # FIX: Use exact equality — "MATCH" in "NO_MATCH" would incorrectly be True
            if ai_decision == "MATCH":
                send_interview_invite(sender)

        mail.logout()  # FIX: Always close the IMAP connection cleanly

    except Exception as e:
        print(f"An error occurred: {e}")


if __name__ == "__main__":
    print("AI Agent Started! Checking inbox every 60 seconds...")
    while True:
        check_inbox()
        time.sleep(60)

How the AI Decision Works

The system prompt constrains Llama 3 to respond with exactly MATCH or NO_MATCH. The prompt is strict by design — LLMs can sometimes add extra words, so the .strip() call removes whitespace. You can expand the criteria in the system_prompt string to match your own job requirements.

3
Packaging

Dockerizing the Agent

If we run the Python script on our local machine, it stops when we close the lid or the terminal. Docker packages your code into a portable "shipping container" that can run anywhere — including your VPS — without installing Python manually.

A

Create requirements.txt

This lists the third-party libraries Docker will install. Note: imaplib, smtplib, email, os, and time are all Python standard library modules — no installation needed.

requirements.txt 
requests
B

Create Dockerfile

This is the build recipe. It uses the official Python 3.11 slim image (minimal size), copies your files, installs dependencies, and sets the startup command.

Dockerfile 
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
COPY agent.py .
RUN pip install -r requirements.txt
CMD ["python", "agent.py"]

Your ai-agent/ folder should now contain exactly three files: agent.py, requirements.txt, and Dockerfile.

4
Deployment

Deploying to Contabo — Going Live

Now we move everything to the cloud. You will need the IP address and root password that Contabo emailed you when you set up your VPS.

1

Connect to your server via SSH

Terminal (your local machine) 
ssh root@YOUR_CONTABO_IP_ADDRESS
2

Install Docker on the VPS

Run these two commands after you are connected via SSH:

Bash (VPS) 
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh
3

Upload Files & Build the Docker Image

Use scp from your local terminal to copy files to the VPS, then SSH back in and build:

Terminal (local → VPS) 
# From your local machine — upload the project folder
scp -r ./ai-agent root@YOUR_CONTABO_IP_ADDRESS:/root/ai-agent

# SSH back into your VPS, then build
ssh root@YOUR_CONTABO_IP_ADDRESS
cd /root/ai-agent
docker build -t my-ai-agent .

Alternative: Use nano to create files directly on VPS

If you prefer, SSH into your VPS and use nano agent.py, nano requirements.txt, nano Dockerfile to paste content directly without SCP.

4

Run the Agent Container

This command starts the container in detached mode (-d), injects your secrets as environment variables (-e), names the container, and sets it to auto-restart on crash or server reboot.

Bash (VPS) 
docker run -d \
  -e EMAIL_ADDRESS="careers@yourdomain.com" \
  -e EMAIL_PASSWORD="your-gmail-app-password" \
  -e CF_ACCOUNT_ID="your-cloudflare-account-id" \
  -e CF_API_TOKEN="your-cloudflare-api-token" \
  --name recruitment-agent \
  --restart always \
  my-ai-agent

Check that the container is running with:

Bash (VPS) 
docker ps                              # Should show recruitment-agent
docker logs -f recruitment-agent       # Tail the live logs

Mission Accomplished — Your AI Agent is Live 🎉

The --restart always flag is the magic trick. If the script crashes or the server reboots, Docker automatically brings the agent back up. You now have an enterprise-grade AI recruitment employee running 24/7 on your own infrastructure, at near-zero cost.
