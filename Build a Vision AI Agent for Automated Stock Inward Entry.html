Build a Vision AI Agent for Automated Stock Inward Entry

Turn photos of handwritten delivery challans and vendor invoices into clean, validated database records — using a Next.js frontend, a FastAPI + PydanticAI backend, and PostgreSQL, all deployed with Docker Compose.

~45 min setup Next.js · FastAPI · PydanticAI Docker Compose · Postgres

In agricultural wholesale and farm operations, incoming stock arrives as chaos — fragmented WhatsApp texts, printed vendor invoices, or handwritten delivery challans scribbled by farm managers. This guide builds a Vision AI agent that lets warehouse staff simply snap a photo of the challan; the agent reads it like an OCR engine, extracts supplier name, product details, batch numbers, and quantities, and enforces a strict data schema before saving it to Postgres.

System Architecture

Built for Agricultural Scale

Separating the UI (Next.js) from the data extraction engine (Python + PydanticAI) gives a scalable foundation that can later expand into nutrient logging, yield prediction, or polyhouse sensor pipelines.

Infrastructure

Docker Compose — Corrected & Secured

The original configuration mapped the FastAPI backend directly to the host with ports: "8000:8000" while claiming it wouldn't be "exposed to the public web" — a direct contradiction. It also hardcoded database credentials and the OpenAI key in plain text. The corrected version below fixes both issues using expose (internal-only) and an external .env file.

Backend

FastAPI + PydanticAI Agent

PydanticAI is a solid choice for structured extraction, but the original code used a deprecated API surface. Current PydanticAI releases use output_type instead of result_type, and result.output instead of result.data. It also imported Optional without using it, and passed images in raw OpenAI dict format instead of PydanticAI's own multimodal content types. All four are fixed below, along with added CORS support so the frontend can actually reach this API.

main.py — Python FastAPI Backend
from fastapi import FastAPI, HTTPException
from fastapi.middleware.cors import CORSMiddleware
from pydantic import BaseModel, Field
from pydantic_ai import Agent, BinaryContent
from typing import List
import base64

app = FastAPI()

# FIX: allow the Next.js container to call this API across the Docker network
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://frontend:3000", "http://localhost:3000"],
    allow_methods=["*"],
    allow_headers=["*"],
)

# ---------------------------------------------------------
# 1. STRICT DATA SCHEMAS — AI must return exactly this shape
# ---------------------------------------------------------
class StockItem(BaseModel):
    product_name: str = Field(description="Generic item name (e.g., Round Purple Brinjal)")
    sku_or_batch: str = Field(description="Batch number or SKU, else 'N/A'")
    quantity_received: int = Field(description="Total count or kg received")
    unit_cost_inr: float = Field(description="Cost per unit in INR, no currency symbols")

class InwardStockManifest(BaseModel):
    supplier_name: str = Field(description="Supplier, farm name, or vendor")
    items: List[StockItem]

# ---------------------------------------------------------
# 2. AGENT — FIX: output_type replaces deprecated result_type
# ---------------------------------------------------------
stock_agent = Agent(
    'openai:gpt-4o',
    output_type=InwardStockManifest,
    system_prompt=(
        "You are an expert warehouse operations assistant. You will be provided with "
        "either raw text or images of handwritten/printed delivery notes. Act as an OCR "
        "and extraction engine: parse the input and extract a clean stock inward manifest. "
        "If unit cost is missing, calculate it from total cost if available."
    )
)

# ---------------------------------------------------------
# 3. ENDPOINT — FIX: BinaryContent instead of raw OpenAI dict
# ---------------------------------------------------------
@app.post("/api/agent/inward-stock")
async def process_inward_stock(raw_input: dict):
    text_content = raw_input.get("text", "")
    image_base64 = raw_input.get("image", None)

    if not text_content and not image_base64:
        raise HTTPException(status_code=400, detail="Provide either text or an image.")

    prompt_input = [text_content or "Analyze this delivery document."]

    if image_base64:
        image_bytes = base64.b64decode(image_base64)
        # FIX: PydanticAI's native multimodal type, not an OpenAI-format dict
        prompt_input.append(BinaryContent(data=image_bytes, media_type="image/jpeg"))

    try:
        result = await stock_agent.run(prompt_input)
        manifest = result.output   # FIX: .output replaces deprecated .data

        # TODO: INSERT INTO stock_logs ... (connect via DATABASE_URL)

        return {"status": "success", "extracted_manifest": manifest.model_dump()}
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

Where Does the Vercel AI SDK Fit In?

This endpoint needs one complete structured JSON block before a database insert, so waiting on a plain fetch is the safer choice here. If you later build a chat interface (e.g., "How much Brinjal did we receive today?"), use the Vercel AI SDK's useChat hook to stream tokens for a ChatGPT-like feel.

Frontend

Next.js Upload Form

The original frontend called a relative path /api/agent/inward-stock, which hits the Next.js server itself — not the separate FastAPI container. It also defined NEXT_PUBLIC_API_URL in Docker Compose but never used it anywhere in the code. Fixed below by actually consuming the environment variable.

app/page.jsx — Next.js Client Component
'use client';
import { useState } from 'react';

export default function StockInwardForm() {
  const [inputText, setInputText] = useState('');
  const [imageBase64, setImageBase64] = useState(null);
  const [result, setResult] = useState(null);
  const [loading, setLoading] = useState(false);

  const handleImageUpload = (e) => {
    const file = e.target.files[0];
    if (file) {
      const reader = new FileReader();
      reader.onloadend = () => {
        const base64String = reader.result.split(',')[1];
        setImageBase64(base64String);
      };
      reader.readAsDataURL(file);
    }
  };

  const handleSubmit = async (e) => {
    e.preventDefault();
    setLoading(true);
    setResult(null);

    try {
      // FIX: call the actual backend URL, not a relative Next.js path
      const apiUrl = process.env.NEXT_PUBLIC_API_URL;
      const response = await fetch(`${apiUrl}/api/agent/inward-stock`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ text: inputText, image: imageBase64 }),
      });

      if (!response.ok) throw new Error(`Server responded ${response.status}`);

      const data = await response.json();
      setResult(data.extracted_manifest);
    } catch (error) {
      console.error("Data extraction failed:", error);
      alert("Failed to process delivery note.");
    } finally {
      setLoading(false);
    }
  };

  return (
    <div className="max-w-2xl mx-auto p-6">
      <h2 className="text-xl font-bold mb-4">Process Delivery Note</h2>
      <form onSubmit={handleSubmit} className="flex flex-col gap-4">
        <div className="p-4 border border-dashed border-gray-400 rounded bg-gray-50">
          <label className="block font-bold mb-2">Upload Photo of Challan/Invoice:</label>
          <input type="file" accept="image/*" onChange={handleImageUpload} />
        </div>
        <textarea
          className="w-full p-3 border rounded border-gray-300"
          rows="4"
          placeholder="Or paste text notes here..."
          value={inputText}
          onChange={(e) => setInputText(e.target.value)}
        />
        <button type="submit" disabled={loading}
          className="mt-2 px-6 py-3 bg-green-700 text-white font-bold rounded hover:bg-green-800 disabled:opacity-50">
          {loading ? 'Analyzing with Vision AI...' : 'Extract & Log Stock'}
        </button>
      </form>

      {result && (
        <div className="mt-8 p-5 bg-green-50 border border-green-200 rounded">
          <h3 className="font-bold text-green-900 mb-2">
            Extraction Successful: {result.supplier_name}
          </h3>
          <ul className="space-y-2">
            {result.items.map((item, index) => (
              <li key={index} className="p-3 bg-white border border-green-100 rounded shadow-sm">
                <strong>{item.quantity_received} units</strong> of {item.product_name}
                <br/>
                <span className="text-sm text-gray-600">
                  Batch: {item.sku_or_batch} | Cost: ₹{item.unit_cost_inr}/unit
                </span>
              </li>
            ))}
          </ul>
        </div>
      )}
    </div>
  );
}

Production

Going Live — Reverse Proxy & Upload Limits

Two things the original guide never mentioned: how the public internet reaches your frontend securely with SSL, and that base64-encoded photos can be large enough to hit default request body size limits.

A

Add Nginx as a reverse proxy with SSL

Run Nginx (or Caddy) in front of the frontend container to terminate HTTPS on your domain, e.g. stock.uvfarms.in, and forward traffic to port 3000 internally.

nginx.conf (excerpt)
server {
    listen 443 ssl;
    server_name stock.uvfarms.in;

    location / {
        proxy_pass http://frontend:3000;
        proxy_set_header Host $host;
    }

    client_max_body_size 15M;  # FIX: allow larger base64 photo uploads
}

Base64 Photos Are ~33% Larger Than the Original File

A 5MB invoice photo becomes roughly 6.7MB as a base64 string inside a JSON body. Without raising client_max_body_size in Nginx and FastAPI's request limits, uploads will silently fail with 413 errors.

Your Vision AI Stock Agent is Production-Ready 🎉

The backend now runs internal-only, secrets live in .env, the frontend actually talks to the right API, and Nginx handles SSL with proper upload limits. Warehouse staff can now snap a photo of any challan and have it land in Postgres as clean, structured data — no manual typing required.
